# tests/evaluation/ — RAG Evaluation Framework

Sprint 5 evaluation pipeline: measures the chatbot's answer quality, retrieval precision, and refusal behaviour against a ground-truth golden dataset using an LLM-as-judge approach.

## Files

| File | Purpose |
|---|---|
| `golden_dataset.json` | 20 ground-truth questions generated by LLM with expected answers and key facts |
| `evaluator.py` | `GroqEvaluator` — LLM judge using `llama-3.3-70b-versatile` via Groq API |
| `metrics.py` | Computes MRR, Source Hit Rate, Avg Judge Score, Pass Rate; generates text report |
| `run_eval.py` | End-to-end runner: calls `/chat` endpoint per question → judges → saves results |
| `results/` | Timestamped JSON result files from past evaluation runs |

## Prerequisites

1. **FastAPI backend running** at `http://localhost:8000` with ChromaDB populated
2. **`GROQ_API_KEY`** set in `.env` (uses Groq's free tier — `llama-3.3-70b-versatile`)

## Running the evaluation

```bash
# From financial-analyst-agent/
uv run python -m tests.evaluation.run_eval
```

Options:
```bash
uv run python -m tests.evaluation.run_eval \
    --dataset tests/evaluation/golden_dataset.json \
    --output  tests/evaluation/results/
```

Results are saved to `results/eval_<YYYYMMDD_HHMMSS>.json`.

## Metrics

| Metric | Description |
|---|---|
| **MRR** (Mean Reciprocal Rank) | Average 1/rank of the first expected source in the returned sources list. 1.0 = expected source always ranked first. Adversarial questions excluded (no expected source). |
| **Source Hit Rate** | Fraction of non-adversarial questions where any expected source appears anywhere in the returned sources list. |
| **Avg Judge Score** | Mean score (0.0–1.0) assigned by the Groq judge across all questions. |
| **Pass Rate** | Fraction of questions where judge score ≥ `PASS_THRESHOLD` (0.6). |

All metrics are reported overall and broken down by question type (`narrative`, `quantitative`, `adversarial`).

## Golden dataset (`golden_dataset.json`)

20 questions across three categories:

| Type | Count | Description |
|---|---|---|
| `narrative` | 15 | Facts from quarterly press releases and the FY25 annual report |
| `quantitative` | 3 | Exact lookups from the stock price CSV |
| `adversarial` | 2 | Out-of-scope questions; correct answer is a clear refusal |

Each question specifies:
- `key_facts`: specific numbers/phrases that MUST appear for a pass
- `expected_sources`: filenames the retriever should return
- `is_refusal_expected`: if `true`, the judge checks for a polite refusal rather than factual correctness

All ground-truth facts were verified directly from the source documents before being written into the dataset — no hallucinated expected answers.

## Judge design

**Factual questions:** The judge checks whether each `key_fact` is present in the chatbot's answer, with explicit tolerance for number-format variation (`"5,076 million"` = `"$5.1 bn"` = `"approximately 5 billion"`).

**Adversarial questions:** The judge checks whether the chatbot explicitly stated it does not have the data, without guessing.

**Deterministic pass/fail:** `PASS_THRESHOLD = 0.6`. The `PASS/FAIL` verdict is computed in Python from the numeric score — the LLM's own verdict field is never trusted because it was found to be inconsistent.

# data/

All on-disk storage for LedgerMind — raw source documents, processed intermediates, vector store, agent memory, and generated exports.

## Directory layout

```
data/
├── raw/                   Source files — PDFs, CSV, XLS (not committed, add manually)
├── processed/             LlamaParse Markdown cache — auto-generated by ingestion pipeline
├── chroma_db/             Persistent ChromaDB vector store — auto-generated by ingestion pipeline
├── exports/               Generated PDF/Excel reports — auto-generated by the agent at runtime
└── agent_memory.sqlite    LangGraph SqliteSaver conversation checkpoints
```

## raw/ — Source documents

Place these files here before running ingestion:

| Filename | Description |
|---|---|
| `ifrs-usd-press-release_q1.pdf` | Infosys Q1 FY26 earnings press release (quarter ended June 30, 2025) |
| `ifrs-usd-press-release_q2.pdf` | Infosys Q2 FY26 earnings press release (quarter ended September 30, 2025) |
| `ifrs-usd-press-release_q3.pdf` | Infosys Q3 FY26 earnings press release (quarter ended December 31, 2025) |
| `ifrs-usd-press-release_q4.pdf` | Infosys Q4 & FY26 full-year earnings press release (quarter ended March 31, 2026) |
| `infosys-ar-25.pdf` | Infosys Integrated Annual Report FY25 |
| `500209.csv` | BSE daily OHLCV stock price history for Infosys (BSE code 500209) |
| `500209.xls` | Infosys investor data workbook (multi-sheet: quarterly financials, ratios, etc.) |

**PDFs are ingested into ChromaDB.** The CSV and XLS are loaded directly as Pandas DataFrames by the quantitative tool at query time — they are not vectorized.

## processed/ — Markdown cache

Auto-populated by `backend/ingestion/pipeline.py`. Each PDF produces a corresponding `.md` file. Re-running ingestion skips PDFs that already have a cached `.md` file, saving LlamaParse API calls.

## chroma_db/ — Vector store

SQLite + binary segment files managed by ChromaDB. Created automatically on first ingestion run. Do not edit manually.

## exports/ — Generated artifacts

PDF reports and Excel workbooks created during agent runs. Files are named `report_<8hex>.pdf` or `analysis_<8hex>.xlsx` and served via `GET /download/{filename}`. Old files are not automatically cleaned up — delete manually if disk space is a concern.

## agent_memory.sqlite — Conversation checkpoints

LangGraph `SqliteSaver` persists the full conversation state for every `session_id`. Allows:
- Multi-turn follow-up questions within a session
- Conversation restoration after a server restart
- History replay via `GET /history/{session_id}`

Delete this file to wipe all conversation history.
